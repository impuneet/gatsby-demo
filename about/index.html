About This Site

rmorabia.com is the personal website of Radhika Morabia.

This is my personal database, which covers my interest in success, freedom, independence, and not being pseudo.

This manifests in a lot of content about the open web, the human self and human rationality, reflections on life, quotes from awesome people, and reviews of everything I've ever consumed.

This all stemmed from a single quote:

What sort of writing could you create if you worked on it (be it ever so rarely) for the next 60 years? What could you do if you started now? Gwern

My goal with this website is to constantly have it grow until one year, five years, fifty years from now, I look back, and I am able to see my own growth through it.

This will all be done through blog posts, and hopefully, by writing a little bit every single day. Unlike blog posts, though, where you hit (write) and run (never edit), most of the writing on this website will be constantly editing old posts.

Everything is a draft. Using a version control system like Git for my source code will allow me to track any and all changes I make at any moment in the future.

This website is coded entirely by me, with help from Jekyll, to help manage posts and allow me to code less, and from the Pure CSS Framework, so that my site will work on any device.

The domain rmorabia.com is currently registered with GoDaddy. (I'm switching to Gandi once my domain expires.) This site and its source code are hosted by Github. This might change in the future, as well, as Github only works for very simple, totally static websites.

Everything I coded is released entirely into the public domain, although Jekyll is using the MIT License and Pure is using the BSD License. I also use the Lato and Merriweather fonts on this website, and those belong to their respected owners.

Why release all my writing, code, design, etc., into the public domain? Why not? If you'd like to quote my writing, or learn from my code, or outright copy my entire website and put it up on another domain, go right ahead! I don't want to deal with the hassle of "Do I have permission?" emails, although I'd appreciate it if you told me what you were using my stuff for.

This website is constantly being changed. If you find an error or have a design suggestion, I'd really appreciate it if you contacted me.
